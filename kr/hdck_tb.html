호두클릭 (탭 버전)

  • 프로그램

    PSD Ai

  • 제작 연도

    2018

  • 역할 (기여도)

    디자이너 (100%)

    기획자 (90%)

회의실 예약 앱인 호두클릭 태블릿 버전으로 제작된 프로젝트입니다. 웹과 앱을 통해 회의실 예약 후 태블릿을 통해 현장에서 사용자가 물리적으로 사용하게 되며, 태블릿은 회의실 입구에 배치합니다. 태블릿 버전은 영문으로 제작 요청이 들어왔고 3가지 모드(OCCUPIED 사용 중, WAITING 체크인 전, AVAILABLE 예약 가능)로 기획되었으며, 색상을 통해 직관적으로 사용자가 확인할 수 있게 디자인되었습니다.

웹 버전 이전에 제작 요청이 들어왔고, 태블릿 버전의 사용목적의 특성상 웹 버전의 느낌과는 다소 다른 느낌의 앱으로 기획과 디자인이 진행되었습니다. 사용자 편리성을 위해 호두클릭 메인 색상을 배제하였고, 호두클릭 가입자 외 일반 사용자 역시 별도 교육 없이 사용할 수 있게 신호등을 연상케하는 색상으로 3가지 모드를 디자인하였습니다.

가이드라인

3가지 모드가 주된 앱으로 별도 교육 없이 모두가 사용 가능하게 제작하는 게 관건이었기에, 모두에게 친숙한 신호등에서 영감을 얻어 색상을 선택하였습니다.

  • image of three circles of each color

    • 빨강 OCCUPIED (사용 중) 첫 번째 모드로 외부인이 태블릿을 확인 후 회의실 입장이 불가하다는 사실을 색상을 통해 인지할 수 있게 빨간색으로 선택하였습니다. 빨간 배경 위 텍스트가 들어가기에 장시간 사용 시 눈의 피로감을 줄 수 있어 채도를 낮추었습니다.

    • 노랑 WAITING (체크인 전) 두 번째 모드로 회의 참석자가 희의 실 도착 시 사용 전 체크인이 필요하다는 신호를 알려주는 모드로 노란색으로 선택하였습니다. 노란색 역시 사용 중 모드에 맞추어 채도가 낮은 노란색으로 선택하였습니다.

    • 초록 AVAILABLE (지금 예약 가능) 세 번째 모드로 회의실이 현재 사용되지 않고 예약이 가능하다는 그린라이트를 밝히는 의미로 초록색을 선택하였습니다. 채도가 낮은 두 모드와 달리 이 모드의 경우 웹과 앱에서만 제어가 가능하여 밝은 계열의 초록을 사용하여 전반적인 디자인에 생기를 부여하였습니다.

  • 성별

    여성 40%, 남성 60%로 남성의 비율이 높고 웹 버전의 성별 서베이와 비슷한 비율로 보입니다. (웹 버전 케이스스터디는 상단 프로젝트에서 확인 가능합니다)

WAITING

체크인 전 모드로 예약 후 예약시간에 맞추어 도착 시 보이는 화면입니다. 이 모드는 예약시간 시작 후 10분 안에 체크인을 해야 회의실 사용이 가능하며, 체크인 없이 10분이 지날 시 예약이 취소됩니다. 3가지 모드 중 중간단계로 노란색으로 선택하였습니다.

회의실에 도착 했을때

체크인 버튼을 클릭하여 예약한 회의실을 바로 사용할 수 있습니다. 체크인 후 모드는 OCCUPIED (사용 중) 모드로 변환됩니다.

image of a web version of calendar

  • 체크인 태블릿 버전 예제 이미지

  • 체크인 버튼
    예약된 회의실은 예약된 시간으로부터 10분 안에 체크인이 필요하며, 체크인이 확인되지 않을 시 예약이 취소됩니다. 버튼 테두리를 활용하여 체크인까지 필요한 시간을 표현하였습니다.
    체크인 버튼 이미지

    • STATUS

      STATUS는 현재 모드를 표시합니다. 모드는 WAITING(체크인 전), OCCUPIED(사용 중) AVAILABLE(예약 가능)이며, 폰트 사이즈를 키워 멀리서도 확인 가능하게 하였으며, 모드에 따라 배경색이 변경됩니다.

    • INFO

      STATUS 모드의 관련된 내용이 출력되며, WAITING의 경우 체크인 까지 남은 시간, OCCUPIED의 경우 예약된 시간이 종료되기까지 남은 시간, AVAILABLE의 경우 다음예약까지 예약이 가능한 시간이 표시됩니다.

    • 레이아웃 이미지

OCCUPIED

"현재 사용 불가"를 의미하는 모드로, 체크인 후 다음 단계 모드입니다. 회사 직원의 회의실 출입을 통제하고, 사용 가능한 예약시간을 알려줍니다. 3모드 중 색상선택에 가장 고민을 많이 한 부분으로 빨간색 특성상 눈의 피로감을 최대한 줄이며 전반적인 디자인의 생기를 많이 놓치지 않은 빨간색으로 선택하였습니다.

회의실이 사용 중 일때

"사용 불가"를 가장 잘 표현하는 색상인 빨간색이 선택되었고, 회의 종료는 권한이 있는 사람만 조기 종료할 수 있습니다.

image of a web version of calendar

  • 노트북을 켜놓고 회의를 하는 사람들의 손이 보이는 이미지입니다.

  • 종료 버튼
    예약한 시간보다 회의가 짧아져 조기 퇴소를 원할 시 종료 버튼을 통해 가능하며, 버튼의 테두리를 사용하여 남은 예약시간을 표현하였습니다.
    image of the check-in button

    • MEETING ROOM

      회의실명이 보이는 영역으로 옵션에 따라 회사명으로 변경 가능하며, 웹 버전 환경설정을 통해 디스플레이 옵션은 변경이 가능하게 기획되었습니다.

    • CURRENT TIME

      현재 시간과 월, 일, 연도를 표시하는 영역으로 사용자가 다른 디바이스를 통해 별도로 시간을 확인하지 않고 태블릿을 통해 바로 확인 가능케 배치하였습니다.

    • 태블릿 버전 레이아웃 이미지로 모든 텍스트가 회색 처리되었고, MEETING ROOM과 CURRENT TIME만 빨간색으로 표시하였습니다.

AVAILABLE

예약이 없는 회의실에 경우 사용되는 모드로 예약이 가능하다는 그린라이트를 보내는 신호로 초록색을 선택하였습니다. 프로젝트 제작 당시 태블릿 버전을 통해 예약 또한 가능하게 기획되었지만, 디자인이 마무리된 후 예약은 웹과 앱을 통해서만 가능하다는 통보를 받아 약간에 변화가 있었습니다. 사전에 사실을 알았다면 초록색이 아닌 다른 색상을 선택하지 않았을까 생각합니다.

회의실이 사용 가능 할 때

회의실 예약이 필요하다면 웹과 앱을 통해 예약 가능합니다.

image of a web version of calendar

  • 노트북으로 웹 버전을 사용하고 있는 사용자의 모습이 담긴 이미지

  • 사용 가능한 시간
    다른 모드와 다르게 버튼이 아닌 시계 형태로 다음 예약까지 예약 가능한 시간이 센터에 보이고 테두리를 통해 예약 가능한 남은 시간이 표현됩니다. 예약은 웹과 앱 버전을 통해 가능하게 기획되었습니다.
    image of the check-in button

    • BUTTON

      중간 정렬된 텍스트와 하얀색 배경 원형으로 디자인된 버튼 영역으로 두꺼운 검정 테두리를 추가하였습니다. 검정 테두리 위 모드 색상과 동일한 색상의 테두리는 모드에 관련된 인디케이터 역할을 하는 테두리로 모드에 적합하게 변경됩니다.

    • SIDE MENU

      예약자 목록 영역으로 예약자의 이름과 부서명, 예약시간이 표시되고 리스트 왼쪽에 화살표를 통해 현재 회의실 사용자를 표시해 줍니다. 목록은 시간대로 정렬되며 초록색 테두리, 폰트 색을 가지고 있는 리스트는 예약이 가능한 시간대를 알려줍니다.

    • image of layout

다른 화면 옵션

데드라인 전 급하게 추가된 디스플레이 옵션 추가 요청으로 두 옵션에 차이가 크지 않습니다. 구비된 장비와 정원 등 메인 디스플레이에서 배제된 정보를 추가하여 기획과 디자인이 진행되었습니다.

  • 옵션 1 디스플레이 옵션 화면

  • 옵션 1

  • 옵션 1 레이아웃 화면

    • 상단

      기본 디스플레이 옵션의 회의실명이 회사명으로 교체하며, 구비된 장비를 아이콘으로 센터에 표시하여 회의실 입장 전 필요한 장비를 확인할 수 있습니다.

    • 메인 컨텐츠

      회의실명과 현재 시간을 상단에 배치하고, 원형 버튼 형식의 모드를 바형과 별도 버튼을 추가하여 디자인하였습니다. 오른쪽 목록 콘텐츠는 메인 디스플레이와 동일합니다.

  • 옵션 2 디스플레이 옵션 화면

  • 옵션 2 레이아웃 화면

    • 메인 컨텐츠

      옵션 1과 동일하게 디자인했습니다. 옵션 1과 다르게 모드 영역이 상하로 가운데 정렬이 되어있어 좀 더 안정감을 제공합니다.
